PPS – Unidad 0 – Tarea

Documentación Principal

Bienvenido/a a la documentación de la Tarea de la Unidad 0 de PPS, cuyo propósito es poner en práctica diversas competencias relacionadas con:

  • Lenguaje de marcas Markdown
  • Sistemas de control de versiones: Git
  • Creación y administración de contenedores: Docker
  • Ciclos de desarrollo software seguros: SecDevOps
  • Elaboración de documentación técnica

Esta documentación recoge cada una de las fases del trabajo, así como las evidencias necesarias (comandos, capturas y explicaciones detalladas) para demostrar el desarrollo completo de la actividad.

Objetivos de la tarea

Esta tarea tiene como finalidad aplicar de forma práctica los contenidos vistos en la unidad, incluyendo:

  • Uso básico y avanzado de Git
  • Automatización mediante GitHub Actions
  • Publicación de documentación con GitHub Pages
  • Generación y despliegue de un contenedor NGINX con Docker
  • Documentación clara siguiendo buenas prácticas DevsecOps

Estructura de la documentación

A continuación se presenta el índice de los documentos que componen esta actividad. Cada sección detalla un proceso concreto siguiendo las especificaciones de la tarea.

Índice

  • Git – Creación del repositorio
    Documentación del proceso de creación del repositorio, configuración inicial, subida de archivos y colaboración.

  • GitHub Actions – Creación del WorkFlow
    Explicación y funcionamiento del workflow que genera la documentación con MkDocs automáticamente.

  • GitHub Pages – Publicación de la documentación
    Configuración de GitHub Pages y despliegue del sitio estático generado.

  • Docker – Contenedor NGINX con la documentación
    Documentación del proceso de creación, levantamiento y pruebas del contenedor NGINX que sirve la página en local.

  • Conclusiones finales
    Reflexiones personales acerca del uso de Git, Docker y SecDevOps.

Descripción general de la actividad

A lo largo de esta práctica se realizaron las siguientes tareas:

  1. Creación del repositorio GitHub y estructura base.
  2. Documentación en archivos Markdown ubicados en docs/.
  3. Desarrollo de un workflow GitHub Actions que automatiza la creación de documentación con MkDocs.
  4. Configuración de GitHub Pages para publicar la web del proyecto.
  5. Creación y despliegue de un contenedor NGINX que sirve la documentación generada utilizando bind-mount desde el repositorio.
